# SAM3 Labeler (PyQt6) — 完整使用手冊

## 給初學者的話

歡迎使用 SAM3 Labeler PyQt6 版本！這是一個協助您在影像上標記物件的桌面應用程式。相較於原始 Gradio 網頁版，PyQt6 版本提供毫秒級的互動回應、原生縮放平移、即時懸停高亮，以及 SAM 推論時不凍結介面等改善。

**這個工具能做什麼？**

- 在影像上以點擊、框選或文字提示三種方式標記物件
- 透過 Meta SAM 3 模型自動辨識物件邊界
- 同時輸出 YOLO OBB / YOLO-Seg / PNG Mask / COCO JSON 四種格式
- 滾輪縮放、中鍵平移，適合高解析度影像的精細標註

## 第一章：安裝

### 1.1 系統需求

| 項目 | 最低需求 | 建議配置 |
|------|----------|----------|
| 作業系統 | Windows 10 / Ubuntu 20.04 / macOS 12+ | Windows 11 / Ubuntu 22.04 / macOS 14+ |
| Python | 3.10 | 3.12 |
| GPU | NVIDIA GTX 1060 (6GB) / Apple M1 | NVIDIA RTX 3060+ (8GB+) / Apple M2+ |
| CUDA | 11.7（macOS 免 CUDA，使用 MPS） | 12.1+ |
| 記憶體 | 8 GB | 16 GB+ |
| 磁碟空間 | 5 GB（含模型） | 10 GB+ |

> **跨平台支援：** 程式會自動偵測最佳運算裝置（CUDA GPU → Apple MPS → CPU），無需手動設定。

### 1.2 安裝步驟

```bash
# 1. 安裝 PyTorch（依您的 CUDA 版本選擇）
# 範例：CUDA 12.1
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

# 2. 安裝相依套件
pip install PyQt6 opencv-python numpy ultralytics
```

### 1.3 下載 SAM 3 模型

SAM 3 模型檔案（`sam3.pt`，約 3.4 GB）不包含在程式中，需手動下載：

1. 前往 [SAM 3 Hugging Face 頁面](https://huggingface.co/facebook/sam3)
2. 申請存取權限（需 Meta 核准）
3. 核准後下載 `sam3.pt`
4. 將 `sam3.pt` 放置於任意位置，啟動時以 `--model` 參數指定

若不想等待審核，可先使用 SAM 2 模型（透過 Ultralytics 自動下載，無需審核）。

### 1.4 啟動程式

```bash
# 基本啟動
python main.py

# 指定模型路徑
python main.py --model /path/to/sam3.pt

# 同時指定圖片與輸出資料夾
python main.py --model /path/to/sam3.pt --images /path/to/images --output /path/to/output
```

程式會直接開啟桌面視窗，無需瀏覽器。

### 2.2 工具列（左側）

三種互動模式，一次只能選一種：

| 工具 | 快捷鍵 | 滑鼠游標 | 說明 |

這是最大的區域，顯示當前影像與標註。

**互動操作：**

| 操作 | 動作 |
|------|------|
| 滾輪上/下 | 以滑鼠為中心放大/縮小 |
| 中鍵拖曳 | 平移畫面 |
| 雙擊左鍵 | 回到適合視窗大小 |
| 按 `F` 鍵 | 回到適合視窗大小 |

**畫面元素：**

- **彩色框線**：每個標註的 OBB 邊界，不同類別不同顏色
- **標籤文字**：帶黑底圓角背景，格式為 `編號. 類別名稱`
- **座標顯示**：左下角顯示目前滑鼠位置的影像座標與懸停標註
- **懸停高亮**：滑鼠移到標註上方時，框線變為虛線
- **選取高亮**：被選取的標註以青色（cyan）粗框顯示
- **推論遮罩**：SAM 執行中時，畫面顯示半透明遮罩與「SAM 推論中...」提示

### 2.4 控制面板（右側，可捲動）

由上而下依序為：

#### 文字分割區

輸入物件名稱（逗號分隔多個），按 Enter 或點「▶ 執行文字分割」。SAM 3 會自動找出所有符合的物件。輸入的名稱會自動新增為類別。

#### 類別區

- **下拉選單**：選擇目前分割要使用的類別
- **+ 按鈕**：新增類別（輸入名稱後按 Enter 或點 +）
- **− 按鈕**：刪除目前選取的類別（有標註使用中的類別無法刪除）

#### 設定區

| 項目 | 說明 |
|------|------|
| **SAM 失敗改用方框** | 勾選後，SAM 無法辨識時以手動框選範圍建立標註 |
| **顯示模式** | 框線 / 遮罩 / 兩者 |
| **多邊形簡化** | 滑桿 0.001~0.020，越小多邊形越精確（頂點越多） |
| **重疊閾值** | 滑桿 0%~50%，0% = 允許重疊，10% = 10% 重疊即禁止 |

#### 輸出格式區

四個勾選框，可同時勾選多個：

| 格式 | 預設 | 說明 |
|------|------|------|
| OBB (旋轉框) | ☑ | `labels/*.txt`，YOLO OBB 訓練用 |
| YOLO-Seg (多邊形) | ☑ | `labels_seg/*.txt`，實例分割訓練用 |
| PNG Mask | ☐ | `masks/*.png`，語意分割用 |
| COCO JSON | ☐ | `coco_annotations.json`，COCO 格式 |

#### 標註列表區

- 每個標註前方有彩色方塊，對應類別顏色
- 支援多選（Ctrl+點擊 或 Shift+點擊）
- 選取後可：
  - **套用類別**：將選中標註改為指定類別
  - **🗑 刪除選中**：刪除選中的標註
  - **清除全部**：刪除所有標註

#### 儲存按鈕

點擊「💾 儲存標註」手動儲存。切換圖片時也會自動儲存。

## 第三章：快速入門

### 步驟 1：準備影像

將所有影像放到同一個資料夾。支援格式：`.jpg`、`.jpeg`、`.png`、`.bmp`、`.webp`。

### 步驟 2：啟動程式

```bash
python main.py --model /path/to/sam3.pt
```

### 步驟 3：載入影像

1. 在「圖片資料夾」欄位輸入路徑（或按 📁 瀏覽）
2. 在「輸出資料夾」欄位輸入儲存路徑
3. 點擊「載入」

成功後會顯示第一張影像。若之前有標註過，會從上次離開的位置繼續。

### 步驟 4：開始標註

1. 確認左側工具列選擇「🖱️ 點擊」
2. 確認右側「類別」下拉選單選對
3. 在影像上點擊物件中心
4. SAM 自動辨識，彩色框出現即為成功

### 步驟 5：切換下一張

按鍵盤 → 鍵，或點左側「下一張 ▶」按鈕。標註會自動儲存。

## 第四章：三種標註方法

### 方法一：點擊分割

**適合情境：** 物件明顯、邊界清楚。

1. 選擇工具「🖱️ 點擊」（或按 `1`）
2. 點擊物件中心
3. SAM 自動辨識邊界

```
    點這裡
      ↓
  ┌───────┐
  │   ●   │  ← 點擊物件中心
  │  瓶子 │
  └───────┘
```

**技巧：** 盡量點在物件正中央。若效果不佳，換個位置再試。

### 方法二：矩形框選

**適合情境：** 物件重疊、AI 判斷不準。

1. 選擇工具「⬜ 框選」（或按 `2`）
2. 在物件左上角按住左鍵
3. 拖曳到物件右下角放開
4. 拖曳過程中會即時顯示橘色虛線矩形

```
  按住拖曳
      ↓
      ●┄┄┄┄┄┄┄┄┐
      ┆ ┌─────┐ ┆
      ┆ │物件 │ ┆  ← 拖曳過程即時顯示
      ┆ └─────┘ ┆
      └┄┄┄┄┄┄┄┄●
              ↑
           放開
```

**與 Gradio 版的差異：** 原版需要點擊兩次定義角落，PyQt6 版支援原生拖曳，操作更直覺。

### 方法三：文字分割

**適合情境：** 一次找出所有特定類型物件。

1. 在右側「文字提示」輸入物件名稱
   - 單一物件：`bottle`
   - 多個物件：`bottle, debris, buoy`（逗號分隔）
2. 按 Enter 或點「▶ 執行文字分割」
3. SAM 3 自動找出所有符合的物件

**注意：** 輸入的名稱會自動新增為類別。推論期間介面不會凍結（背景執行緒處理）。

## 第五章：編輯標註

### 5.1 選取標註

1. 切換到「✋ 選取」工具（或按 `3`）
2. 點擊標註框 → 該標註變為青色粗框
3. 再次點擊同一標註 → 取消選取

**多選方式：**

| 方式 | 操作 |
|------|------|
| 逐個點擊 | 每點一個標註，加入或移出選取集合 |
| 拖曳框選 | 在空白處按住拖曳，框內的標註全部加入選取 |
| 列表多選 | 在右側標註列表中 Ctrl+點擊 或 Shift+點擊 |
| 全選 | Ctrl+A |
| 取消全選 | Esc |

### 5.2 刪除標註

1. 選取要刪除的標註
2. 按 `Delete` 鍵或點「🗑 刪除選中」
3. 標註消失

### 5.3 修改類別

1. 選取要修改的標註
2. 在「套用類別」旁的下拉選單選擇正確類別
3. 點擊「套用類別」

### 5.4 清除全部

點擊「清除全部」按鈕，移除目前影像的所有標註。

## 第六章：類別管理

### 新增類別

**方法一：** 在右側輸入框輸入名稱，按 Enter 或點 `+`

**方法二：** 使用文字分割時，輸入的名稱自動新增

### 刪除類別

1. 在下拉選單選擇要刪除的類別
2. 點 `−` 按鈕
3. 若有標註使用該類別，會提示無法刪除

**限制：** 至少需保留一個類別。

### 切換類別

每次標註前確認類別正確。使用下拉選單切換。

## 第七章：顯示模式

在右側設定區切換三種模式：

### 框線模式（預設）

只顯示 OBB 邊界框線。適合確認邊界位置。

### 遮罩模式

顯示半透明彩色填充。適合確認標註覆蓋範圍。

### 框線 + 遮罩模式

同時顯示框線與填充。資訊最完整。

## 第八章：縮放與導覽

### 滾輪縮放

以滑鼠所在位置為中心進行縮放。

| 操作 | 效果 |
|------|------|
| 滾輪向上 | 放大 12% |
| 滾輪向下 | 縮小 12% |

縮放範圍：5% ~ 3000%

### 中鍵平移

按住滑鼠中鍵拖曳，移動畫面。游標會變為抓手圖示。

### 適合視窗

以下操作可將影像回復為適合視窗大小：

- 按 `F` 鍵
- 雙擊左鍵

## 第九章：儲存與輸出

### 自動儲存

切換影像（← → 鍵、上一張/下一張按鈕、跳轉）時自動儲存。

### 手動儲存

按 `Ctrl+S` 或點「💾 儲存標註」。

### 輸出檔案結構

```
輸出資料夾/
├── images/              ← 已標註的影像副本
│   ├── frame_001.jpg
│   └── frame_002.jpg
├── labels/              ← OBB 格式標註 (class x1 y1 x2 y2 x3 y3 x4 y4)
│   ├── frame_001.txt
│   └── frame_002.txt
├── labels_seg/          ← YOLO-Seg 多邊形格式
│   ├── frame_001.txt
│   └── frame_002.txt
├── masks/               ← PNG 遮罩（若有開啟）
│   └── frame_001.png
└── classes.txt          ← 類別清單
```

### 輸出格式說明

| 格式 | 檔案位置 | 座標格式 | 用途 |
|------|----------|----------|------|
| OBB | `labels/*.txt` | `class x1 y1 x2 y2 x3 y3 x4 y4`（正規化） | YOLO OBB 模型訓練 |
| Seg | `labels_seg/*.txt` | `class x1 y1 x2 y2 ... xn yn`（正規化多邊形） | YOLO-Seg 模型訓練 |
| Mask | `masks/*.png` | 像素值 = 標註索引+1，背景=0 | 語意分割 |
| COCO | `coco_annotations.json` | COCO 標準 JSON | COCO 格式訓練 |

## 第十二章：與 Gradio 版本的差異

### 效能差異

| 操作 | Gradio 版 | PyQt6 版 |
|------|-----------|----------|
| 點擊 → 畫面更新 | 300-500 ms（HTTP 來回 + 伺服端重繪） | < 5 ms（本地 QPainter） |
| 懸停高亮 | 不支援 | < 3 ms |
| 框選拖曳 | 無即時回饋（需兩次點擊） | 即時橘色矩形 |
| 20 個標註重繪 | 20× alpha blending + JPEG 編碼 | 向量繪製 + 遮罩快取 |
| 滾輪縮放 | 不支援 | 矩陣變換 |
| 中鍵平移 | 不支援 | 原生支援 |
| SAM 推論期間 | 整個介面凍結 | 背景執行緒，UI 不凍結 |

### 操作差異

| 功能 | Gradio 版 | PyQt6 版 |
|------|-----------|----------|
| 框選方式 | 點兩次定義角落 | 原生拖曳 |
| 模式切換 | 左側 Radio 點擊 | 左側 Radio + 數字鍵 1/2/3 |
| 全選/取消 | 底部按鈕 | Ctrl+A / Esc |
| 瀏覽資料夾 | 手動輸入路徑 | 📁 按鈕開啟系統對話框 |
| 介面存取 | 瀏覽器開啟 localhost:7860 | 直接桌面視窗 |

### 新增功能

- 座標即時顯示（左下角）
- 懸停顯示標註名稱
- 標籤文字帶黑底背景
- SAM 推論中顯示等待遮罩
- 中鍵平移
- 雙擊適合視窗

## 第十三章：完整操作範例

### 範例：標註海洋廢棄物影像

**情境：** 您有 500 張海岸監視器截圖，要標註 plastic_bottle、float_ball、foam、other_debris、oilspill 五類。

1. **啟動程式**
   ```bash
   python main.py --model /home/ncu_mporg/sam3.pt
   ```

2. **載入影像**
   - 圖片資料夾：`/home/ncu_mporg/marine_debris/images`
   - 輸出資料夾：`/home/ncu_mporg/marine_debris/output`
   - 點「載入」

3. **設定類別**
   - 輸入 `plastic_bottle`，按 Enter
   - 輸入 `float_ball`，按 Enter
   - 依此類推新增五個類別

4. **快速批次標註**
   - 在文字提示輸入 `plastic_bottle, float_ball, foam, other_debris`
   - 按 Enter 執行文字分割
   - SAM 3 自動找出物件（推論中介面不凍結）
   - 檢查結果，刪除錯誤標註，修正類別

5. **精細補標**
   - 按 `1` 切回點擊模式
   - 選擇正確類別
   - 點擊文字分割遺漏的物件

6. **切換下一張**
   - 按 → 鍵
   - 上一張自動儲存

7. **發現標錯了**
   - 按 `3` 切到選取模式
   - 點擊標錯的標註
   - 按 `Delete` 刪除
   - 重新標註

8. **高解析度影像**
   - 滾輪放大到小物件
   - 中鍵拖曳移動到目標區域
   - 標註完按 `F` 回到全景

9. **完成**
   - 到輸出資料夾查看 `labels/` 和 `labels_seg/` 目錄
   - 使用附帶的腳本訓練 YOLO OBB 模型

## 第十四章：常見問題

### Q1：點擊物件沒有反應？

- 確認工具選的是「🖱️ 點擊」（按 `1`）
- 確認左下角座標有顯示（表示滑鼠在影像範圍內）
- 若 SAM 無法辨識，改用「⬜ 框選」模式
- 勾選「SAM 失敗改用方框」作為保底

### Q2：程式顯示「SAM 推論中...」很久？

- SAM 3 模型較大，首次推論需載入模型，可能需要 10-30 秒
- 後續推論通常 1-3 秒
- 推論期間可觀察進度條，介面不會凍結

### Q3：忘記選類別就標註了？

1. 按 `3` 切到選取模式
2. 點擊該標註
3. 在右側選擇正確類別
4. 點「套用類別」

### Q4：圖片太大看不清楚？

- 滾輪放大（以滑鼠為中心）
- 中鍵拖曳平移
- 按 `F` 或雙擊回到全景

### Q5：程式當掉，標註會遺失嗎？

只要切換過影像，之前的標註都已儲存。重啟後載入同一資料夾即可繼續。

### Q6：如何與原始 Gradio 版共用標註資料？

兩個版本的標註格式完全相同（labels/*.txt 為 YOLO OBB 格式），輸出目錄可互通。

## 第十五章：名詞解釋

| 名詞 | 解釋 |
|------|------|
| **OBB** | Oriented Bounding Box，旋轉邊界框，以四個頂點表示 |
| **Seg** | Segmentation，分割，以多邊形頂點表示物件精確邊界 |
| **Mask** | 遮罩，二值化影像表示物件位置（0=背景, 255=物件） |
| **COCO** | Common Objects in Context，常用的影像標註 JSON 格式 |
| **SAM** | Segment Anything Model，Meta 開發的通用分割模型 |
| **QPainter** | Qt 框架的繪圖引擎，直接在視窗上繪製向量圖形 |
| **正規化座標** | 座標值為 0~1 之間，表示相對於影像寬高的比例 |
